## Codebase Patterns
- Build with `npm run build` (tsc), typecheck with `npm run typecheck` (tsc --noEmit)
- Entry point is `src/index.ts` with shebang `#!/usr/bin/env node`
- Compiled output goes to `dist/` directory
- Uses Node16 module resolution (ESM-compatible with .js extensions in imports)
- @types/node is needed for `process` global; added as devDependency
- No external runtime dependencies; uses built-in `fetch` for HTTP
- Arg parsing lives in `parseArgs`; flags checked against a `known` Set, unknown flags exit non-zero
- Mutually-exclusive flags (e.g. --beta/--next) are validated in `parseArgs` and exit 1 on conflict
- npm version lookups hit `https://registry.npmjs.org/n8n/{tag}`
- n8n GitHub release tags use format `n8n@{version}` via `/repos/n8n-io/n8n/releases/tags/{tag}`
- Responses are validated for shape before use; errors thrown with status/context
- `main().catch` prints `Error: {message}` to stderr and exits 1
